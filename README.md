# E-commerce Sales Analysis (SQL)

## Overview
This project analyzes a public UK Online Retail dataset using SQL to evaluate revenue trends, customer purchasing behavior, and product performance.

## Dataset
- 20,036 transaction records
- 705 unique customers
- Transaction-level sales data
- Publicly available dataset used for analytical practice

## Objective
To derive business insights from transactional e-commerce data using SQL-based analysis.

---

## Key Business Questions

- What is the total revenue generated?
- How does revenue change month-over-month?
- What is the average order value (AOV)?
- Who are the top revenue-generating customers?
- What percentage of revenue comes from top customers?
- What is the repeat purchase rate?
- What is the churn rate (90-day inactivity)?
- Which products generate the highest revenue?

---

## Key Insights

- **Total Revenue:** 420,227.75  
- **Average Order Value:** 448.48  
- **Repeat Purchase Rate:** ~20%  
- **Churn Rate (90-day inactivity):** ~70%  
- **Top 10 customers contribute:** ~20% of total revenue  
- Revenue shows significant month-to-month fluctuation

## Technical Skills Demonstrated

- SQL Aggregations (SUM, COUNT, GROUP BY)
- CTE (Common Table Expressions)
- Window Functions (LAG)
- Conditional Aggregation
- Revenue & Customer Behavior Analysis
- Churn & Retention Analysis
